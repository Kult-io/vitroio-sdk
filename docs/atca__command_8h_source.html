81 // this is the ATCACommand parameter structure. The caller to the command method must
82 // initialize param1, param2 and data if appropriate. The command method will fill in the rest
83 // and initialize the packet so it's ready to send via the ATCAIFace.
84 // this particular structure mimics the ATSHA and ATECC family device's command structures
85 
86 // Note: pack @ 2 is required, @ 1 causes word alignment crash (though it should not), a known bug in GCC.
87 // @2, the wire still has the intended byte alignment with arm-eabi. this is likely the least portable part of atca

101  // used for transmit/send

103 
104  //--- start of packet i/o frame----

107  uint8_t param1; // often same as mode

110  // command + crc (see: x08 verify data1 + data2 + data3 + data4)
111  // this is an explicit design trade-off (space) resulting in simplicity in use
112  // and implementation
113  //--- end of packet i/o frame
